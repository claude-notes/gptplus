# Codex 安装交付服务独立页面设计

## 目标

在现有站点中新增一个路径为 `/codex/` 的独立静态页面，用于展示 Codex 安装交付服务。页面沿用 `https://qimuai.cn/codex/` 的业务内容、信息结构和价格体系，并以相近的深色首屏、青绿色强调色、浅色内容分区和卡片式套餐布局重新实现。

新页面必须与现有首页及其 SEO/GEO 体系隔离。现有首页、文章页、全局样式、全局脚本、站点地图、爬虫规则、LLM 文件和 GEO 分析文件均不修改。

## 非目标

- 不重做或调整现有首页。
- 不修改现有页面的标题、描述、结构化数据、canonical、hreflang 或其他 SEO 元数据。
- 不修改 `robots.txt`、`sitemap.xml`、`llms.txt` 或 `GEO-ANALYSIS.md`。
- 不复用参考站的 CSS、JavaScript、图片或其他远程资源。
- 不增加表单、支付、登录、外部 API 或用户信息提交功能。
- 不调整 Codex 服务的套餐价格和主要交付内容。

## 文件与隔离方式

所有新页面资源放在独立目录中：

- `codex/index.html`：页面结构、独立元数据和正文内容。
- `codex/style.css`：页面专用视觉样式和响应式规则。
- `codex/script.js`：移动导航、二维码弹窗和必要的交互。
- `codex/wechat-qr.jpg`：用户提供的微信二维码原图。

新页面不加载现有的 `css/style.css` 或 `js/script.js`，避免任何双向样式或行为耦合。页面不依赖构建工具，保持与当前静态站点一致的部署方式。

## 页面信息结构

页面依次包含以下内容：

1. 顶部导航：品牌标识、为什么选我们、服务内容、套餐价格、FAQ、GPT Plus。
2. 深色首屏：服务定位、599 元起提示、主要行动按钮、四项服务亮点、设备交付示意卡片。
3. 为什么选我们：不用反复试错、避开共享账号风险、装完马上验收。
4. 服务内容：设备与系统检查、账号与 Plus 准备、Codex 多端安装、任务跑通与手机指挥。
5. 套餐价格：
   - 基础安装包：599 元/次。
   - 标准跑通包：999 元/次，并作为视觉主推套餐。
   - 进阶工作流包：1999 元/次。
   - 团队小班包：4999 元起。
6. 上门服务和 Plus 权益说明。
7. 交付完成标准与直接退款条件。
8. 安全边界：屏幕共享、账号授权、敏感项目隔离。
9. FAQ：沿用参考页的六项常见问题。
10. 微信预约咨询：设备信息说明和大尺寸微信二维码。
11. 简洁页脚。

页面文案沿用参考页的 Codex 安装交付服务内容，不替换成现有 GPT Plus / Pro 购买业务。

## 视觉设计

视觉基调为可信、清晰、偏技术服务型：

- 顶部导航为白色背景和细边框，滚动时保持可见。
- 首屏使用接近黑色的深绿背景，配合低对比度设备界面装饰。
- 主要强调色使用青绿色，价格提示和少量状态点使用暖黄色。
- 内容区在白色和浅灰绿色之间交替，形成明确的阅读节奏。
- 卡片使用细边框、克制圆角和轻阴影，不使用大面积玻璃拟态或紫色 AI 渐变。
- 标准跑通包使用深色卡片和青绿色按钮，与其他套餐形成明显层级。
- 动画仅限轻微浮动按钮提示、元素入场和悬停反馈；在 `prefers-reduced-motion` 下关闭非必要动画。

页面布局以约 1180 像素的内容宽度为基准。桌面端首屏使用左右双栏，套餐使用四列；平板和手机端按可读性依次收敛为双列或单列。

## GPT Plus 与微信二维码交互

采用已确认的方案 C：顶部导航入口与右下角浮动按钮并存。

- 顶部导航中的“GPT Plus”使用青绿色强调样式，点击后打开二维码弹窗。
- 页面右下角常驻“微信咨询”浮动按钮，点击后打开同一弹窗。
- 首屏的“预约远程安装”和各套餐按钮跳转到页面底部微信咨询区，不触发外部跳转。
- 弹窗显示用户提供的完整二维码原图、标题“添加微信，预约 Codex 安装”和简短操作提示。
- 弹窗支持关闭按钮、点击遮罩关闭和按 `Escape` 关闭。
- 弹窗打开后锁定页面滚动，并将键盘焦点移动到关闭按钮；关闭后恢复触发入口的焦点。
- 手机端浮动按钮缩短为紧凑样式，二维码图片按视口缩放且保持完整，不裁切关键图案。
- 页面底部咨询区再次显示大尺寸二维码，确保无需打开弹窗也能直接扫码。

如果 JavaScript 未加载，顶部“GPT Plus”和浮动按钮均退化为跳转至底部咨询区，核心内容和二维码仍然可访问。

## 新页面元数据与现有 SEO/GEO 隔离

`codex/index.html` 仅设置该页面自己的基础元数据：

- 中文页面语言和移动端 viewport。
- 与 Codex 安装交付服务匹配的 `title` 和 `description`。
- 指向 `/codex/` 的 canonical。
- 基础 Open Graph 标题、描述和页面类型。

不为此次任务修改任何现有页面的元数据，也不把新页面加入现有 sitemap、llms 或 GEO 文件。新页面内不新增会改变站点既有 GEO 规则的专用内容格式。

## 响应式与无障碍

- 桌面端导航显示完整内部锚点；手机端使用菜单按钮展开导航。
- 菜单按钮通过 `aria-expanded` 和可读标签表达状态。
- 二维码弹窗使用 `role="dialog"`、`aria-modal="true"` 和标题关联。
- 所有可点击控件保留清晰的键盘焦点样式。
- 正文颜色、按钮颜色和深浅背景满足清晰阅读所需的对比度。
- 二维码图片提供明确的替代文本，不将二维码作为唯一的咨询说明。
